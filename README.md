# The-Gaming-Room-Client
The Gaming Room was the client for this project, and their goal was to expand their Android-based game, Draw It or Lose It, into a web-based application that could run across multiple platforms. The game required support for multiple teams and players, synchronized gameplay, and the ability to scale in a distributed environment. My role was to design the software architecture and evaluate operating platforms to ensure the application could meet these requirements while remaining efficient and user-friendly.

One aspect I did particularly well was the evaluation of operating platforms. I provided a clear analysis of Linux, Windows, macOS, and mobile devices, highlighting their advantages, weaknesses, and licensing costs. This gave the client a strong foundation for making informed decisions about deployment. I also organized the documentation into logical sections, which made the design easy to follow and professional in presentation.

Working through the design document was especially helpful because it required me to think holistically about the system before writing any code. By identifying requirements, constraints, and architectural considerations early, I was able to anticipate challenges such as memory management, distributed communication, and cross-platform compatibility. This planning process streamlined development and reduced the risk of overlooking critical functionality.

If I were to revise one part of my work, I would improve the domain model section by adding more detailed UML diagrams and examples of class interactions during gameplay. This would make the design more visual and accessible to stakeholders. I would also expand on scalability strategies, such as containerization and load balancing, to demonstrate how the system could handle thousands of concurrent users.
